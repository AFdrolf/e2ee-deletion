\begin{abstract}
    A common component of modern end-to-end encrypted (E2EE) applications is that of \emph{deletion of messages}. Applications support this via a variety of features, such as erasing conversations, and recently via more advanced features such as unsending messages, dissappearing messages, etc. Despite the ubiquity of such features, however, message deletion in E2EE applications is poorly understood: Do these features actually erase messages? What security guarantees should we expect of such features?

    This work initiates the formal study of message deletion in E2EE applications. 
    - Landscape, threat model, etc.
    - Attacks
    - Definitions


    Our analysis uncovers a broken ecosystem: We show attacks against all message deletion features in WhatsApp, allowing full recovery of deleted text messages, media attachments, and more. 

    
\end{abstract}
